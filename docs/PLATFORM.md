## Architecture Overview

```
  ESP32 Buoy (sensors)
      │  LoRa 868 MHz
      ▼
  LoRa Gateway (GW001–GW003)
      │  HTTP POST or MQTT
      ▼
  Gateway Bridge (gateway_bridge.py)
      │  SQLite offline queue + retry
      ▼
  FastAPI Backend (apps/api/)
      │  PostgreSQL + PostGIS (telemetry, sensors, sites, alerts)
      │  ML inference (WHS + RF + IF)
      │  WebSocket broadcast
      ▼
  React GIS Dashboard (src/)
      Leaflet maps, real-time charts, alert management
```

### Data Source Tags

Every record in the system carries a `source` field that is **never alterable** after ingestion:

| Source       | Meaning                                                    |
|:-------------|:-----------------------------------------------------------|
| `iot`        | Real sensor data received via gateway/MQTT                 |
| `manual`     | Operator-entered field reading (always quality_flag=suspect)|
| `simulation` | Generated by the built-in simulator (demo only)            |
| `import`     | Bulk CSV import                                            |
| `cached`     | Served from local cache when API is offline                |
| `offline`    | Sensor reported offline / no recent data                   |

## Prerequisites & Setup

### Windows Native (No Docker)

**Required:**
- Python 3.11+ (tested on 3.13.13)
- PostgreSQL 16+ with PostGIS extension
- Node.js 20+
- Git

**PostgreSQL / PostGIS Setup:**
```powershell
# Start PostgreSQL service (run as Administrator)
net start postgresql-x64-18

# Enable PostGIS (done automatically by seed script)
psql -U postgres -c "CREATE EXTENSION IF NOT EXISTS postgis;"
```

**One-click setup:**
```powershell
.\scripts\setup.ps1
```

**MQTT (optional — for real IoT ingestion):**
```powershell
# Run as Administrator:
.\scripts\setup-mqtt.ps1
```

## Running the Platform

### Start FastAPI Backend
```powershell
.\scripts\run-api.ps1 -Reload      # Development with hot reload
.\scripts\run-api.ps1 -Production  # Production (4 workers)
```
- API: http://localhost:8000
- Docs: http://localhost:8000/docs
- Health: http://localhost:8000/health

### Start React Dashboard
```powershell
.\scripts\run-web.ps1
```
- Dashboard: http://localhost:3000

### Run Telemetry Simulator
```powershell
# Sends realistic sensor data every 5 seconds
python -m apps.api.scripts.simulator --interval 5

# Specific sensors only
python -m apps.api.scripts.simulator --sensors AQ001,AQ002
```

### Run Gateway Bridge (with serial)
```powershell
# Simulate mode (no hardware needed)
python -m apps.api.gateway_bridge --simulate

# Real serial port
python -m apps.api.gateway_bridge --port COM3
```

## MQTT / Gateway Guide

### Topic Hierarchy
```
aquasentinel/{org_id}/{gateway_id}/telemetry   → sensor readings (uplink)
aquasentinel/{org_id}/{gateway_id}/status      → gateway heartbeat (uplink)
aquasentinel/{org_id}/{gateway_id}/health      → gateway diagnostics (uplink)
aquasentinel/ack/{gateway_id}                  → delivery ack (downlink)
aquasentinel/commands/{gateway_id}/{sensor_id} → sensor commands (downlink)
```

### Gateway Authentication
In development mode, gateway auth is disabled.
In production, include headers:
```
X-Gateway-Id: GW001
X-Api-Key:    <gateway_api_key>
```

### Packet Deduplication
Packets are deduplicated by SHA-256 hash of the full payload body.
If a packet is received twice (gateway retry), the second is silently discarded.
The `sequence_no` field is used for ordering.

## ML Models Disclaimer

> ⚠️ **CRITICAL: All ML scores in this system are prototype models trained on synthetic data. They are NOT validated for operational decisions.**

| Score | Model | Training Data | Validated? |
|-------|-------|--------------|------------|
| Water Health Score (WHS) | Rule-based formula | N/A (deterministic) | No — not official WQI |
| Flood Risk Score | RandomForest (v1.0-prototype) | Synthetic (3000 samples) | **NO** |
| Pollution Anomaly Score | IsolationForest (v1.0-prototype) | Synthetic (5000 samples) | **NO** |

**Never use these scores for:**
- Official flood warnings or evacuation orders
- Regulatory water quality compliance
- Public health advisories
- Media communications about water safety

Always follow official bulletins from CWC, IMD, TNPCB, and local authorities.

## Data Sources & Integrity

### Source Labelling Rules
1. All API responses include a `source` field
2. Manual entries are always tagged `source=manual` and `quality_flag=suspect`
3. Simulation data is tagged `source=simulation`
4. Mock mode (`VITE_MOCK_MODE=true`) shows a yellow ⚠️ DEMO badge on all data
5. IoT data older than 15 minutes is automatically tagged `is_stale=true`

### Quality Flags
| Flag | Meaning |
|------|---------|
| `good` | Passed all plausibility checks |
| `suspect` | One or more checks triggered (e.g. pH < 4 or > 10, tilt > 45°) |
| `bad` | Critical fault (buoy capsized, multiple sensor failures) |
| `missing` | No reading received |
